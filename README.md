# SAP BTP CAP – Procure-to-Pay (P2P) Project
### ABC Manufacturing Pvt. Ltd. | Module: SAP MM | Platform: SAP BTP

---

## Project Overview

This is a full-stack SAP BTP CAP (Cloud Application Programming) project implementing the **Procure-to-Pay (P2P)** business process for a fictitious company, ABC Manufacturing Pvt. Ltd.

The application covers the complete P2P cycle:

```
Purchase Requisition → Purchase Order → Goods Receipt → Invoice Verification → Payment
```

---

## Tech Stack

| Layer         | Technology                          |
|---------------|-------------------------------------|
| Backend       | SAP CAP (Node.js)                   |

## Entities (Database Tables)

| Entity                  | SAP Equivalent Table | Description                        |
|-------------------------|---------------------|------------------------------------|
| Vendors                 | LFA1, LFB1          | Vendor Master                      |
| Materials               | MARA, MARC, MARD    | Material Master                    |
| PurchaseRequisitions    | EBAN                | Purchase Requisitions (ME51N)      |
| PR_Items                | EBAN (items)        | PR Line Items                      |
| PurchaseOrders          | EKKO                | Purchase Order Header (ME21N)      |
| PO_Items                | EKPO                | Purchase Order Items               |
| GoodsReceipts           | MKPF                | Goods Receipt Header (MIGO)        |
| GR_Items                | MSEG                | Goods Receipt Items                |
| Invoices                | RBKP                | Invoice Header (MIRO)              |
| Invoice_Items           | RSEG                | Invoice Line Items                 |

## Business Actions

| Action              | Description                                              |
|---------------------|----------------------------------------------------------|
| `approvePR`         | Approve a Purchase Requisition                           |
| `convertPRtoPO`     | Convert approved PR to Purchase Order with vendor        |
| `postGoodsReceipt`  | Post GR (Mvt Type 101) – updates stock automatically     |
| `verifyInvoice`     | Run Three-Way Match (PO + GR + Invoice) and verify       |
| `getP2PSummary`     | Get dashboard KPIs (total PRs, open POs, spend, etc.)   |

## Local Setup & Run

### Prerequisites
- Node.js >= 18
- SAP CDS CLI: `npm install -g @sap/cds-dk`

### Steps
```bash
# 1. Clone / extract the project
cd p2p-cap-project

# 2. Install dependencies
npm install

# 3. Run locally (SQLite in-memory)
npm run dev

# 4. Open in browser
# OData Service: http://localhost:4004/p2p
# Fiori Preview: http://localhost:4004
```

## SAP BTP Deployment

```bash
# 1. Build MTA archive
mbt build

# 2. Login to BTP Cloud Foundry
cf login -a https://api.cf.<region>.hana.ondemand.com

# 3. Deploy
cf deploy mta_archives/p2p-cap-project_1.0.0.mtar
```

## API Endpoints (OData V4)

| Endpoint                              | Description                    |
|---------------------------------------|--------------------------------|
| `GET /p2p/PurchaseOrders`             | List all Purchase Orders       |
| `GET /p2p/PurchaseRequisitions`       | List all PRs                   |
| `GET /p2p/GoodsReceipts`             | List all Goods Receipts        |
| `GET /p2p/Invoices`                   | List all Invoices              |
| `GET /p2p/Vendors`                    | List all Vendors               |
| `GET /p2p/Materials`                  | List all Materials             |
| `POST /p2p/approvePR`                 | Approve a PR                   |
| `POST /p2p/convertPRtoPO`             | Convert PR to PO               |
| `POST /p2p/postGoodsReceipt`          | Post Goods Receipt             |
| `POST /p2p/verifyInvoice`             | Three-Way Match + Verify       |
| `GET /p2p/getP2PSummary()`            | Dashboard Summary              |
